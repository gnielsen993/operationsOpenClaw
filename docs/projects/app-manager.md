# App Manager Overview

## Purpose
App Manager is the operating layer for app work: ideas, app progress, TestFlight prep, release decisions, and follow-up.

## Operating model
- Gabe owns final approval for all public ships.
- TestFlight is the safe proving ground for changes.
- The App Work folder is the source of truth for active work.
- GitHub is the public-safe mirror and overview surface.
- Jarvis may draft additions, changes, and deletions, but must show what changed and why.

## App surfaces
- **Strada** — Gabe-owned; marketing drafts are allowed, sending requires approval.
- **FitnessTracker** — Gabe-owned; product/release work routes through Gabe.
- **GameKit** — lighter workflow while in development.
- **DesignKit** — shared foundation for most apps except Strada.

## Cadence
- Daily triage
- Weekly review
- Release checkpoint before every TestFlight or public ship

## Progress tracking
Each active app should show:
- current status
- current goal
- next action
- blockers
- what changed
- what was completed
- campaigns / dockets / experiments

## Safety rules
- Public ships require approval.
- Deletions must be explained.
- Security and safety issues are hard stops.
- TestFlight can be used to validate changes before release.
