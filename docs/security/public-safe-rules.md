# Public-Safe Rules

This repo may be public, so it follows stricter rules than the private OpenClaw workspace.

## Never commit
- API keys, tokens, passwords, private keys, `.p8` files, secrets env files
- Telegram/chat IDs, private phone numbers, personal addresses
- Raw transcripts, raw memory files, or private daily notes
- Unredacted App Store Connect/Firebase/GitHub credentials
- Sensitive project/client names unless explicitly approved for public reference

## Allowed
- High-level capability descriptions
- Sanitized workflow instructions
- Public GitHub repo links
- Public app/product status
- Security policies and operating rules

## If unsure
Do not commit the detail. Replace it with a generic description and keep specifics in the private OpenClaw workspace or Obsidian vault.
